## 🔴 Критические проблемы (требуют исправления)

### 1. Использование /tmp без защиты от symlink attack в tor.sh
**Файл:** `modules/tor.sh`  
**Строка:** `grep -v "^ExitNodes\|^StrictNodes" "$TOR_CONFIG" > /tmp/torrc.tmp`  
**Проблема:** Файл создаётся в /tmp без `mktemp`, что позволяет атакующему создать symlink и перезаписать произвольный файл.  
**Риск:** Symlink attack, race condition, запись в произвольные файлы.  
**Решение:** Использовать `mktemp -d` с `chmod 700` (как в relay.sh).

### 2. rm -f /var/lib/dpkg/lock* без проверки в menu.sh
**Файл:** `modules/menu.sh`  
**Строка:** `run_task "Чистка пакетов" "rm -f /var/lib/dpkg/lock* && dpkg --configure -a 2>/dev/null || true"`  
**Проблема:** Удаление lock-файлов без проверки что dpkg/apt не запущены.  
**Риск:** Повреждение базы данных пакетного менеджера, concurrent write corruption.  
**Решение:** Проверять что dpkg не запущен перед удалением lock-файлов.

### 3. eval в run_task (потенциальная уязвимость)
**Файл:** `modules/core.sh`  
**Строка:** `eval "$@"` в функции `run_task`  
**Проблема:** Если переменные берутся из пользовательского ввода без валидации — возможен command injection.  
**Риск:** Выполнение произвольных команд.  
**Примечание:** В menu.sh переменные проходят валидацию через `_validateDomain`, `_validatePort`, `_validateUrl`, что снижает риск. Однако рекомендуется избегать eval где возможно.

## 🟡 Средние проблемы (рекомендуется исправить)

### 4. SSH порт меняется без бэкапа
**Файл:** `modules/security.sh`  
**Строка:** `sed -i "s/^#\?Port [0-9]*/Port $new_ssh_port/" /etc/ssh/sshd_config`  
**Проблема:** Нет бэкапа конфига перед изменением.  
**Риск:** Потеря доступа к серверу при ошибке.  
**Рекомендация:** Добавить `cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.$(date +%s)` перед sed.

### 5. Отсутствие валидации в tor.sh при вводе страны
**Файл:** `modules/tor.sh`  
**Функция:** `changeTorCountry()`  
**Проблема:** Пользовательский ввод страны не валидируется через регулярное выражение.  
**Риск:** Некорректная конфигурация ExitNodes.  
**Рекомендация:** Добавить `[[ "$country" =~ ^[A-Z]{2}$ ]]` перед записью.

### 6. shell=True в некоторых командах web_panel.py
**Файл:** `modules/web_panel.py`  
**Проблема:** Некоторые команды в словаре COMMANDS используют строковый формат (что приводит к shell=True):  
```python
"xray_test": (f"xray -test -config {XRAY_CONFIG} 2>&1", "read"),
"warp_ip": ("curl -s --connect-timeout 8 -x socks5://127.0.0.1:40000 https://api.ipify.org 2>/dev/null || echo 'ERROR'", "read"),
```
**Риск:** Хотя команды фиксированы (не из пользовательского ввода), формат с shell=True менее безопасен.  
**Рекомендация:** Перевести на `["bash", "-c", '...']` формат.

### 7. Отсутствие rate limiting на SSH перед сменой порта
**Файл:** `modules/security.sh`  
**Функция:** `changeSshPort()`  
**Проблема:** Нет проверки что порт не занят другим сервисом.  
**Риск:** Конфликт портов.  
**Рекомендация:** Добавить `ss -tlnp | grep ":$new_ssh_port "` перед применением.
